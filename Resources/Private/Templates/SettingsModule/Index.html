AI Content Generator

Configure the AI providers used by the AI button on content elements. Text and image generation use separate credentials, so you can mix providers freely (e.g. OpenRouter for text, OpenAI for images).

Text Generation

Used to fill text, select and collection fields

API key of OpenAI or any OpenAI-compatible provider.

Examples: gpt-4o, gpt-4o-mini, claude-sonnet-5 (via a compatible gateway)

OpenAI: https://api.openai.com/v1/chat/completions
OpenRouter: https://openrouter.ai/api/v1/chat/completions
Ollama (local): http://localhost:11434/v1/chat/completions

Sent with every text generation request to steer the AI's behaviour and tone.

Image Generation

Used to create images for image fields — stored in fileadmin/ai_generated/

Separate key for the image provider. Leave empty to disable image generation.

OpenAI: gpt-image-1, dall-e-3
OpenRouter: openai/gpt-image-1, google/gemini-2.5-flash-image

OpenAI: https://api.openai.com/v1/images/generations
OpenRouter: https://openrouter.ai/api/v1/images
OpenRouter (chat-based): https://openrouter.ai/api/v1/chat/completions — uses modalities: ["image","text"]; pick an image-capable model.

Examples: 1024x1024, 1536x1024 (landscape), 1024x1536 (portrait)

How to use

  1. Save your settings above (API keys, models, endpoints).
  2. Open the Web > Page module.
  3. Click the AI button in the header of any content element.
  4. Select the fields to fill — including image fields — write your instruction and hit Generate.
  5. Review the generated content and images, then press Fill to save. Images are stored in fileadmin/ai_generated/.
